Generalidades

Esta herramienta, accesible tanto desde el menú Herramientas como desde TOPOGRAFÍA  → ÚTILES, permite procesar archivos procedentes de láser-scan con varios millones de puntos.


El objetivo de la herramienta es procesar información en paquetes de datos para poder realizar futuras triangulaciones, edición de nodos topográficos, generación de líneas de nivel, etc.

Este menú permite contraer o expandir las subventanas mediante el icono .



No hay ninguna limitación en cuanto al número de puntos a tratar por parte del programa, aunque sí cierta dependencia del hardware del ordenador en cuanto a su potencia.

Al seleccionar esta herramienta, aparece el cuadro de diálogo mostrado en la imagen superior.

Los pasos a seguir son:
  1. Declarar el tipo de datos (formato de los ficheros y extensión).
  1. Ejecutar el análisis de los puntos a partir de una lista con los nombres de los ficheros que contienen los datos.
  1. Crear la base de datos, según una rejilla.
  1. Visualización de los datos según diferentes criterios (monocromo, reflectancia o cota).
  1. Filtrar la información para aligerar la base de datos y eliminar puntos no deseados.
  1. Usar diversos útiles que permiten obtener, por ejemplo, curvas de nivel y/o un archivo de puntos con los datos ya filtrados y que puede ser de interés en otras partes del programa.



Tipos de datos

En la parte dedicada al TIPO DE DATOS se puede seleccionar la extensión de los ficheros ASCII que contienen la información así como su formato.

La extensión del archivo la indica el usuario con hasta 4 caracteres, siendo por defecto .xyz. En cuanto al formato, se trata de ficheros de puntos donde al menos deben existir 3 columnas con las coordenadas X,Y,Z de cada punto, aunque también puede haber columnas previas a las coordenadas que se corresponden, por ejemplo, con el número de punto, la reflectancia, o cualquier otro parámetro. Así, por ejemplo, si se escoge como formato uno de la forma a b c X Y Z, entonces el programa espera uno o varios archivos de 6 columnas, correspondiéndose las 3 primeras con algún parámetro del punto y las 3 últimas con sus coordenadas. Los diferentes formatos ascii que se describen admiten también como separadores de las columnas además de los espacios en blanco " ", las comas "," y el punto y coma ";".

El programa admite también otros varios formatos de archivo de nube de puntos: .las/.la20, .laz, .pts, .e57...

El formato para los ficheros .pts tiene que ser uno de los dos siguientes:
  • X Y Z REFLECTANCIA RED GREEN BLUE
  • X Y Z REFLECTANCIA


En el caso de cargar un fichero .las sin color real pero tener una imagen de fondo, el programa permitirá colorear los puntos cargados con la imagen, de tal forma que la imagen proyecte a cada punto su color. Para poder realizar este proceso deberemos tener cargada la imagen y clicar en la casilla Colorear con imagen (se explica en el apartado de Útiles).

Permite cargar ficheros ascii de extensión .pts: Tienen en una primera línea el número total de puntos, y a continuación un punto en cada línea con siete valores: X Y Z intensidad(reflectancia) R G B.



En Tipo de datos podemos usar algunas opciones adicionales:
  • dX, dY, dZ: si los puntos vienen en coordenadas relativas o desplazados, aquí podemos declarar los distintos incrementos que se sumarán a esas coordenadas.
  • [Ver cabecera]: muestra la cabecera del fichero para que el usuario pueda identificar el formato de estos archivos.
  • Saltar n lineas: ignora las primeras n lineas del archivo, en caso de que estas no se correspondan con datos de puntos (encabezados, etc).
  • Reflectancia:  La reflectancia mide la capacidad de las superficies para reflejar la luz. Es uno de los datos que puede venir en este tipo de archivos. Si se corresponde con la primera columna, entonces es recomendable activar esta opción para que el programa la tenga en cuenta a la hora de leer los datos y pueda así ser usada en las opciones de visualización.
  • Corregir la escala en los archivos .las/.laz: Algunos archivos tienen valores de escala erroneos, error que podemos corregir activando esta casilla.


Análisis

Antes de efectuar el análisis propiamente dicho, es preciso crear una lista con los nombres de todos los archivos de puntos que conforman el conjunto de los datos.
Se tiene la posibilidad de crear la lista de los archivos de entrada mediante el selector de archivos, incluso pueden estar en una carpeta diferente a la carpeta de trabajo. En este caso se crea siempre la lista ISPOL.lt1



Esta lista debe crearse aunque los datos estén contenidos en un único archivo. Para ello se pulsa sobre el botón [Crear lista], apareciendo entonces un cuadro de diálogo en el que simplemente se debe declarar el nombre del archivo que contendrá la lista. Una vez creado el fichero (de extensión .lt1), el programa añadirá automáticamente a la lista todos los archivos con la extensión especificada en el tipo de datos que sea capaz de encontrar en el directorio de trabajo. Por otro lado, el botón [Editar] muestra el contenido del archivo para que se mueda modficar añadiendo o quitando elementos de la lista.

Una vez creada la lista, ya es posible procesar todos los puntos pulsando sobre el botón [Analizar]. El programa analiza los archivos de la lista, determinando el número total de puntos y los valores máximos y mínimos de X, Y, Z y reflectancia.


Base de datos

Tras analizar la nube de puntos al completo, es el momento de crear una base de datos con toda la información obtenida. El objetivo de esta base de datos es reorganizar toda la información una vez procesada y almacenarla para que el usuario pueda recuperarla en cualquier momento sin necesidad de volver a analizar todos los datos de nuevo.

Al crear esta base de datos los puntos se agrupan según una cuadrícula regular. El número de celdas creadas se calcula según el parámetro pmin, que es el promedio mínimo de puntos por celda, es decir, el mínimo número medio de puntos que habrá en cada celda (es el mínimo porque el programa tiene establecido un número de celdas máximo).

Las celdas se muestran de diferentes colores en función del número de puntos que contiene, lo que permite distinguir a simple vista las distintas densidades de puntos tomados por el escáner láser.

La base de datos se genera por defecto en una subcarpeta de la carpeta de trabajo con el nombre laser, aunque el usuario puede especificar otro nombre si lo desea.

Una vez generada la base de datos, es posible recuperarla en cualquier momento pulsando sobre el botón [Abrir], lo que evita tener que efectuar el análisis de los puntos de nuevo en otra sesión de trabajo con el programa.

El botón [Cerrar] cierra todos los ficheros de datos de la base de datos activa, lo que libera memoria una vez finalizado el trabajo. Este paso es necesario si se desea cargar otra base de datos a continuación.

El programa, al generar una base de datos laser, extrae de los archivos .las la siguiente información:
  • Coordenadas del punto X,Y,Z
  • Reflectancia
  • Color real (r,g,b). Si no viene este valor se le puede asignar a partir de una imagen de la zona con la utilidad Colorear con imagen.
  • Clasificación: Los puntos pueden ser clasificados por categorías, por ejemplo: Terreno, Vegetación baja, Vegetación media, Vegetación alta, Construcciones, Agua, Puntos Clave, Ruido, etc... Las 64 primeras categorías están reservadas, pero se pueden clasificar los puntos hasta en 256 categorías diferentes.
Pulsando el icono el programa cuenta automáticamente los puntos que hay en la base de datos de cada uno de los tipos.

Tanto en la visualización como para la generación de un modelo del terreno (líneas de nivel, perfiles, etc) se pueden activar/desactivar distintas categorías. Por ejemplo pueden desactivarse todas dejando solo activos los puntos de Terreno.
El estado de activación de la clasificación se guarda/recupera en el archivo de configuración (ispol.cfg) además de la opción de ver siempre la rejilla.




Visualización

Una vez creada la base de datos, es posible visualizar en pantalla los datos usando distintos criterios:


  • Color por clase: Los puntos se colorean según su clasificación.

  • Color por reflectancia: Los puntos se colorean según la reflectancia.
  • Color por cota: Los puntos se colorean según su cota.
  • Color real. Los puntos se colorean según el colorido RGB que haya definido en el fichero (o el que definamos usando Colorear con imagen.
Si la cantidad total de puntos es muy grande, la representación de los mismos podría ser demasiado lenta. Para evitar este problema, es posible indicar un número máximo de puntos a visualizar por cada celda o por metro cuadrado, y pulsar [Ver puntos] para visualizar solo esos. Si se pulsa sobre el botón [Ver todos], entonces se muestran en pantalla la totalidad de los datos.

Si pulsamos sobre [OpenGL] podremos ver el contenido de la base de datos laser en una ventana en 2D o 3D y en tiempo real (opción por defecto). La base de datos es pasada "entera" a memoria para poder ser dibujada con celeridad.

La visualización 3D se configura con el menú que se muestra al pulsar el botón [OpenGL]:



Las opciones de este menú son:
  • Ver Todos / Ver Puntos: Indican si se van a considerar todos los puntos de la base de datos o sólo un cierto número de puntos por celda.
  • Pmáx: Indica el número de puntos máximo a leer de cada celda en caso de que se seleccione la opción "Ver Puntos".
  • Color por clase / reflectancia / cota / Color real: indica el modo de coloreado de los puntos de la base de datos. La reflectancia o el color real deben estar disponibles en la base de datos laser.
  • OpenGL 2D: Muestra los puntos en dos dimensiones bien en la ventana integrada o en una ventana independiente.
  • OpenGL 3D: Muestra los puntos en tres dimensiones bien en la ventana integrada o en una ventana independiente.
  • Ver cartografía: Permite ver la cartografía en la ventana OpenGL.
  • Ventana integrada: muestra la visualización de los puntos, bien en 2D o en 3D, en la propia ventana de Ispol.
  • Ventana independiente: muestra la visualización de los puntos, bien en 2D o en 3D, en una ventana independiente.

  • Distancia engordar puntos: Indica la distancia en metros, medida desde la cámara al punto de la geometría que se mira (centro de la ventana) a partir del cual, al acercarse la cámara a dicho punto, los puntos se "engordan" para dar una mejor sensación de continuidad de la información.

  • Información al mover el cursor: Al activar esta casilla, el programa nos da la información de cada punto simplemente con situar el cursor del ratón encima de cada uno de ellos. La información que muestra es relativa a coordenadas, color, reflectividad y clase. Si no se activa esta casilla, en la ventana OpenGL podremos ver esta información en la esquina superior izquierda clicando con el botón izquierdo del ratón encima de cada punto.

  • Descargar los datos OpenGL al cerrar ventana: Indica si al cerrar la ventana de OpenGL se desea que se descargue de memoria la información de la base de datos automáticamente (no ocurre nada con la base de datos laser en sí). Para otra visualización será necesario volver a leer esa información.

  • Descargar datos OpenGL ¡ya!: Permite liberar la memoria ocupada por los puntos de la base de datos leidos para la visualización. Esta opción sólo está disponible si se han leido datos y se ha cerrado la ventana sin descargar información (no ocurre nada con la base de datos laser en sí). Si esta información queda en memoria hasta que se libere con este botón, cerrando la ventana de OpenGL con la opcion de "Descargar" marcada o cuando se salga de Istram.

  • Zona a representar en OpenGL: Permite acotar entre qué coordenadas XY se van a visualizar los puntos. Podemos escoger las coordenadas tecleandolas directamente, usando el ratón (Seleccionar celdas para ver) o seleccionar todos los puntos usando Seleccionar todas las celdas.
La navegación por esta visualización es igual que en "Vista 3D".
  • Botón izquierdo del ratón: selecciona el punto al que se mira
  • Rueda del ratón: acerca o aleja la cámara a los puntos.
  • Rueda del ratón, usada como botón, o botón central, junto con arrastre del ratón: permite que la cámara gire alrededor del punto que está en el centro de la imagen.
  • Botón derecho del ratón junto con arrastre del ratón: permite desplazar la cámara sobre los puntos.


Filtros

Es posible filtrar la base de datos con el objetivo de eliminar puntos que aportan información poco relevante o errónea al conjunto total y así aligerar el proceso.



Hay que tener en cuenta que, llegados a este punto, los datos se encuentran ordenados según distancias al escáner y ángulos acimutales y cenitales (para cada ángulo cenital, habrá varios barridos láser con distintos ángulos cenitales). Teniendo esto presente, los posibles criterios de filtrado son:
  • Número máximo de puntos por celda (Pmáx): Es un criterio similar al de Visualización, es decir, el programa se queda como máximo con el número aquí indicado por cada una de las celdas de la rejilla. El filtrado de puntos se hace de manera alterna, de tal forma que si, por ejemplo, en una celda hay 50.000 puntos y se indica que como máximo se admiten 5.000, el programa se queda con un dato de cada diez de forma alternativa.
  • Contorno: Permite seleccionar un recinto (una polilinea cerrada). Si está activa la casilla [ ]Mismo tipo, se seleccionan también los recintos que usen el mismo tipo de linea. Este filtro se queda solo con los puntos que estén dentro/fuera del recinto según esté seleccionada la opción Dentro/Fuera. Si se emplea la opción Separar, se crea una nueva base de datos con los puntos contenidos en cada recinto, generándose el archivo .las correspondiente. Si se selecciona el tipo Filtrar Dentro el programa utiliza el resto de filtros, operando solo con los puntos que están dentro del recinto o de los recintos del tipo especificado. Los puntos que están fuera no se alteran.
  • Rango de cotas: Todos los puntos que no estén dentro del rango [Zmín,Zmáx] son desechados. Esto es útil para no considerar aquellos puntos que han sido tomados por alguna circunstancia especial (por ejemplo, un ave que sobrevolaba la zona).

  • dZ Rasante DR/IZ: Desecha todos los puntos que no estén contenidos en [Cota de rasante +/- dZ]. Solo utilizable cuando se abre esta ventana Nubes densas de puntos teniendo la pestaña Rasantes de Obra Lineal abierta.
  • Cotas aisladas: Se eliminan aquellos puntos de una zona cuya cota difiera en más del valor indicado en dZ respecto al punto más cercano, es decir, elimina 'picos', tal y como se muestra en la siguiente imagen:

  • Puntos principales: Permite eliminar puntos alineados que, aunque sus coordenadas sean correctas, realmente no aportan información relevante al conjunto (puntos alineados). Este filtrado se hace según la tolerancia lateral indicada.
  • Piramidal: Este filtro es equivalente al filtro de triángulos en topografía. Viene definido por un dZ mínima, dZ máxima y una Distancia máxima.

  • Limpiar terreno: Elimina todos los puntos que superen una cierta inclinación de talud con respecto a los puntos de su entorno. Muy útil para eliminar las bóvedas de un túnel, edificaciones, árboles, postes, vehículos, etc.

  • Archivo .ENP: Permite cargar un archivo .enp de comandos de filtrado para llevar a cabo varias operaciones sucesivamente.
Al lado de cada una de las opciones hay una casilla en la que poner el número de una de las clasificaciones, o bien 0. Si se pone el número de una de las clasificaciones, al filtrar lo que hará será colocar esos puntos filtrados en la clasificación indicada. Si se pone 0, los puntos filtrados sencillamente se eliminarán de la base de datos.

Regenerar la base de datos al terminar: si tras filtrar la base de datos queda un número de puntos muy inferior al original, es conveniente activar esta opción para que vuelva a crear una distribución de celdas más acorde con la nueva densidad de puntos. Se puede elegir que se quede solamente con una clase en concreto, o poner 0 en la casilla de Clase a guardar para guardarlas todas.

[Filtrar a:] aplica los filtros seleccionados, generándose entonces una nueva base de datos en la carpeta especificada en el campo adyacente. Si en ese campo se indica el mismo nombre que la base de datos original, se mostrará una advertencia informando de que se sustituirá la base de datos original por la nueva filtrada y pidiendo la confirmación del usuario.

[Informe] muestra un archivo con información sobre el número de puntos antes y después del filtrado, los filtros aplicados y el número de puntos eliminados con cada uno de ellos.



Útiles

En esta sección se proporcionan una serie de utilidades básicas para usar la nube de puntos ya procesada:

Esta sección posibilita la generación de curvas de nivel, según una equidistancia en cota a partir de la casilla Crear curvas de nivel. Por otra parte, mediante el comando Crear curvas de nivel suavizadas el programa genera internamente una base de datos sobre una rejilla que puede ser regular o no. Permite crear isolineas sobre esta nueva base. Se generan las isolieas sin necesidad de triangularla, con lo que se acelera el proceso. Aparece un cuadro que veremos más abajo clicando en la casilla Crear curvas de nivel.Suavizada.



  • Crear curvas de nivel: Genera curvas de nivel a partir de la nube de puntos, usando la equidistancia entre curvas especificada en la casilla adyacente al botón. Cuando en Filtros está activo Contorno y se ha seleccionado una línea válida, las líneas de nivel generadas se crearán sólo Dentro/Fuera de ese recinto o de todos los del [ ]Mismo tipo.

  • Crear curvas de nivel. Suavizada: Mediante esta opción tenemos la posibilidad de suavizar la cota de los puntos utilizando los de alrededor pero sin modificar su posición original, y luego crear curvas de nivel sobre una rejilla. Se requiere la siguiente información:
    • Equidistancia entre curvas de nivel: igual que en la opción anterior.
    • Distancia entre puntos: La nueva base de datos esta formada por puntos situados sobre una rejilla regular y aqui se da la resolución de la rejilla. Esta distancia debe ser mayor que la distancia promedio entre puntos de los datos originales.
    • Distancia de busqueda. Cada punto nuevo utiliza para calcular su cota la de los puntos situados dentro de esta distancia.
    • Coeficiente de peso y curva de pesos. Se utiliza una curva de Gauss para dar un mayor peso a los puntos más próximos y menos a los mas alejados.
Cuando en Filtros está activo Contorno y se ha seleccionado una línea válida, las líneas de nivel generadas se crearán sólo Dentro/Fuera de ese recinto o de todos los del [ ]Mismo tipo.

El color, reflectancia y clasificación se heredan del punto más próximo. En un principio la base de datos obtenida tiene las mismas celdas que la original, de modo que si la densidad de puntos es mucho menor conviene guardar un archivo .las y volver a crear con este la base de datos.
Al crear curvas de nivel con esta nueva base de datos se obtienen líneas más suaves que con la base original, pues utilizando parámetros adecuados se puede eliminar bastante ruido.


  • Añadir puntos (Símbolos activos): añade a la base de datos los puntos (símbolos activos) presentes en la cartografía.

  • Triangular desde un punto: Esta utilidad crea una malla de lineas EDM, al estilo 3D caras que permite extraer transversales con la opción de extraplomos en paredes pseudoverticales. La malla se crea triangulando los puntos interiores a un contorno utilizando como coordenadas los angulos horizontal y vertical vistos desde el punto definido, en lugar de las coordenadas (x,y). Es posible limitar la Máxima longitud de malla indicando un valor en metros.
Activando la casilla correspondiente los resultados se guardan en un archivo .ttg.

  • Insertar otra N.D.P: Permite sustituir en el interior de uno o varios contornos, del mismo tipo, los puntos de la N.D.P actual, por puntos de otra N.D.P. Los contornos deben estar previamente definidos en la pestaña Filtros, activando el filtro por Contorno, seleccionando un contorno y marcando, si es el caso, la opción [ ]Mismo tipo. Al pulsar sobre [ ] Insertar otra N.D.P permite seleccionar la N.D.P que va a insertar sus puntos. 
  • Sumar Z: Incrementa en un valor fijo (positivo o negativo) la cota de todos los puntos de la nube densa de puntos.
  • .xyz/.las/.tif/.ttp: Abre un menú desplegable con el cual realizar diversas operaciones de guardado de archivos:

    • Guardar .xyz / .las / .laz: Guarda un archivo .xyz, .las o .laz con la base de datos activa.

    • Rasterizar: Genera una imagen raster a partir de los puntos de la base de datos laser. Nos preguntará el nombre del fichero a generar, su resolución en m/pixel y su formato (Geotiff, ASC,...)

    • .ttp simplificado: genera un archivo .ttp (fichero de triangulación) a partir de una base de datos laser, dando el número máximo de puntos y unos parámetros iniciales para un filtrado piramidal (dZmin, dZmax y Dist máx). El programa filtra la base de datos y genera un archivo .las; si el número de puntos de éste todavía supera a los previstos, crea una nueva base de datos con este archivo .las, incrementa los parámetros de filtrado y repite el proceso hasta conseguir una base con un número de puntos inferior al previsto. En este momento genera el archivo de triangulación .ttp y guarda también esta superficie como triangulación actual.

    • Guardar .las/.xyz -> CRS destino/HUSO destino/transformación coordenadas: guarda un archivo .las o .xyz aplicando la transformación especificada en el CRS de destino, o en el huso de destino, o en Herramientas/Transformación de coordenadas.
  • Editar datos: Esta herramienta permite editar puntos de la base de datos en grupo o individualmente. Primero se ha de seleccionar una zona de trabajo, en la cual podemos visualizar los puntos de cualquiera de las maneras habituales (por clase, por reflectancia, por cota, color real).

    • Cambiar de clase y borrar: Permite seleccionar/deseleccionar utilizando un cursor circular de radio variable. Los elementos seleccionados se pueden cambiar de clase en bloque a una clase concreta o se pueden borrar.

    • Editar un punto: permite seleccionar los puntos de uno en uno para editar diversos atributos: coordenadas, reflectancia, color RGB, clase o incluso borrarlos.

    • +/-: junto a los campos RGB y Ref., estas casillas nos permiten especificar una tolerancia para las herramientas de rango.

    • Rango RGB/REF: selecciona todos los puntos cuyo color/reflectancia estén dentro del rango del valor del punto actual más (menos) la tolerancia especificada en las casillas +/-.

    • Cambiar clase: cambia la clase del punto a la clase activa en la lista de la izquierda. La clase activa es la seleccionada en la columna Destino.

    • Deshacer: el botón deshace los cambios que se hayan hecho a las propiedades del punto seleccionado.

    • Aplicar cambios a la base de datos: acepta las ediciones que hemos llevado a cabo y las incorpora de manera definitiva a la base de datos.
    • Crear curvas de nivel: el botón el programa nos permite generar curvas de nivel de la base de datos en la zona de trabajo cargada.
  • Colorear con imagen: da color (RGB, "color real") a los puntos de la base de datos laser a partir del color de una imagen de fondo que tenemos que tener cargada.

  • Suavizar cotas: Mediante esta opción tenemos la posibilidad de suavizar sólo la cota de los puntos utilizando los de alrededor pero sin modificar su posición original. Permite por tanto suavizar la base de datos laser. Se requiere la siguiente información:
    • Distancia entre puntos: La nueva base de datos esta formada por puntos situados sobre una rejilla regular y aqui se da la resolución de la rejilla. Esta distancia debe ser mayor que la distancia promedio entre puntos de los datos originales.
    • Distancia de busqueda. Cada punto nuevo utiliza para calcular su cota la de los puntos situados dentro de esta distancia.
    • Coeficiente de peso y curva de pesos. Se utiliza una curva de Gauss para dar un mayor peso a los puntos más próximos y menos a los mas alejados.
El color, reflectancia y clasificación se heredan del punto más próximo. En un principio la base de datos obtenida tiene las mismas celdas que la original, de modo que si la densidad de puntos es mucho menor conviene guardar un archivo .las y volver a crear con este la base de datos.
Al crear curvas de nivel con esta nueva base de datos se obtienen líneas más suaves que con la base original, pues utilizando parámetros adecuados se puede eliminar bastante ruido.
  • Cubicar: Permite cubicar entre dos bases de datos laser, con la opción de hacerlo solo en un recinto o también de crear los recintos de desmonte y terraplén. Crea una tercera base de datos sobre una rejilla regular, con la diferencia de cotas de las otros dos bases.
  • Banda por PK: Si estamos dentro de Obras lineales, en Planta o Alzado y accedemos al cuadro de diálogo de Nubes Densas de Puntos, en la pestaña de Útiles disponemos de la herramienta [Banda x Pk]. Esta herramienta permite seleccionar ejes y PKs sobre ellos, numéricamente o gráficamente  definiendo una banda por semiancho en PKs y en distancia al eje, de modo que la visualización de puntos normal (2D o 3D) o si están cargados en OpenGL, se ciña a esa banda. Si vamos picando PKs y esta abierta la ventana OpenGL, el cambio de visualización es inmediato.
Se puede seleccionar un archivo .per diferente. Este archivo de salida será igual que el ISPOLn.per más los puntos de la NDP en el tramo seleccionado.
La herramienta añadir a ISPOLx.per va recorriendo los perfiles de este archivo y ajusta automáticamente el semiancho en PK de la Banda para obtener como máximo el número de puntos que se le indica para cada perfil, los proyecta sobre el PK teniendo en cuente la pendiente longitudinal y les asigna un símbolo con el tamaño indicado y un tipo que se toma de un rango de símbolos en función de la intensidad del color original. Estos símbolos se pueden ver tanto en el editor de perfiles como en un dibujo de perfiles transversales.
Si se activa la opción de Crear Superficie se crea una superficie cerrada uniendo los puntos de la nube en el perfil, para ello hay que definir el tipo de la superficie y configurar una referencia angular para ordenar los puntos, así podremos realizar mediciones con esta superficie.



Ver en Sección Real: Si se activa, al desplazar el cursor por el eje entre los PKs definidos, en la ventana de la Sección se incorporan los puntos de la Nube Densa, y podemos editar estos perfiles.

La opción Volcar a contorno genera con el recinto de la nada entre Pks una línea y la selecciona como contorno para filtrar.

Es posible añadir a la Tabla de puntos para visualizar en la rasante y en el dibujo del perfil longitudinal los puntos de la nube que se proyectan en el eje. Para acceder a esta opción desde el menú Alzado.

Desde este cuadro también se acceden a las mismas utilidades que permiten alterar la cota de entidades gráficas: Obtener cota, lineas a cota, símbolos a cota...

  • Longitud máxima de malla: Cuando se usa una nube de puntos como superficie MDT, y en ese menú se escoge usar la nube de puntos para cortar por mallas, en vez de por triángulos, podemos especificar aquí una longitud máxima de malla. Al triangular, filtrará los triángulos para que no haya mallas mayores que la permitida.
Edición de datos 3D

Selección de puntos volumétrica a través de una esfera o de una caja. Se puede posicionar cualquiera de ellas manualmente a traves de coordenadas globales o situando el volumen con un clic en el visor de nubes densas de puntos. Se puede editar el volumen, en el caso de la esfera especificando el radio y en el caso de la caja especificando ancho, largo y alto.







Bases de datos cargadas

Vamos a poder trabajar con hasta 5 bases de datos laser simultaneamente. Cuando se genera una base de datos nueva y se lee, su información pasa a la base de datos actual.

Cuando se carga una basa de datos desde la ventana inferior, su información se lee y pasa a ser la base de datos activa.



Una superficie MDT puede estar definida con cualquiera de las cinco bases de datos laser disponibles, y se pueden cortar
perfiles transversales a varias bases de datos laser simultaneamente.

Mediante las casillas Atenuación y Actual podré disminuir la visualización dando un porcentaje al fichero Laser cargado como actual y al resto.

Los botones [Iniciar], [Guardar] y [Cargar] permiten inicializar, salvar y recuperar respectivamente las superficies declaradas en el proyecto mediante el uso de archivos .cnd.

Posibilidad de tener simultaneamente hasta 100 bases de datos de nubes densas de puntos.
