Generalidades

Los elementos de un escenario no están "sueltos" sin ningún control. Están organizados y clasificados según su tipo y función, como se muestra en el árbol de organización jerarquica del escenario. Los nodos organizadores pueden tener algunas propiedades configurables.

Los organizadores trabajan como carpetas en las que se almacena o hace referencia a los elementos de la escena.
Hay organizadores fijos puestos por Virtual 3D para poder encontrar en ellos ciertos tipos de datos necesarios para configurar la escena (nodos subraiz). Hay organizadores que se generan para mantener juntos elementos generados automáticamente y que tienen una vinculación especial (nodos de obra, ejes o grid). Hay organizadores que puede crear el usuario para clasificar la geometría como desee (carpetas). Y hay organizadores que dan una calificación específica a los elementos que contienen (tren de objetos).


Subraiz

Estos nodos son inalterables, deben existir con su nombre y contener aquello que les caracteriza: Geometría, Cámaras, Luces, Entorno, Render y Animaciones. No hay acceso a sus propiedades, al pulsar con el cursor sobre ellos, se accede al primer nodo hijo.


Carpeta

Es un organizador de caracter general, sólo se pueden crear dentro de Geometría. Se les puede dar un nombre.

Hay una carpeta especial predefinida dentro de Geometría: "Sólidos 3D" a esta carpeta irán, por defecto, todos los objetos que se creen, se carguen o se importen en Virtual 3D.

Cuando se entra en Virtual 3D desde el menú de Alzado de Obra lineal y se muestra la geometría de los ejes activos, se generan automáticamente un conjunto de carpetas que organizan la obra. Así se genera una carpeta principal para la obra, que se ubica directamente en el nodo "Geometría", con el nombre del proyecto.

En ella se pude indicar que se envíen las lineas de borde de obra, generadas con la geometría de la misma, a la cartografía. Tambien se puede controlar que se pinten o no las marcas viales del proyecto, a veces pueden ser tantas que relenticen demasiado el trabajo con el visor 3D.

Dentro de la carpeta de obra, se genera una carpeta por cada eje y acuerdo. Cada una de ellas contiene las geometrías que se generan a partir de la información de dichos ejes, pero también otras carpetas conteniendo cada viaducto y las marcas viales. Esas marcas viales a su vez se clasifican en carpetas según el típo de línea que las genera.

En las carpetas de eje se dispone de opciones para ubicar la cámara en un punto concreto de un eje definido por su PK, el código y la distancia a dicho código.

También se generan carpetas especiales cuando se cargan grid de datos topográficos, ya sean grid binarios o ttps que  tienen un número de triángulos muy grande.


Tren de objetos

Este tipo de organizador no contiene a otros elementos, simplemente hace referncia a ellos y a la relación de contiguidad que se establece entre ellos. Su utilidad es agupar actores que van a representar el mismo papel: objetos que van a seguir una mismo recorrido.
